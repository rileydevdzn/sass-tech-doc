Writing in GitHub: Adding Images and Visual Content

Introduction

This guide is for adding images and other visual content to your README files. This guide assumes you already have a basic knowledge of:

  • Using headings
  • Styling text with bold and italic
  • Adding links

If you're not familiar with these, you can check out our GitHub Writing Basics page.

Adding images

There are 3 common ways to add images to your READMEs.

  • Markdown syntax ![]()
  • The <img> element
  • The <picture> element

Markdown syntax ![]()

You can display an image using !, then wrapping the alt text in brackets [], and wrapping the link for the image in parentheses ().

![Octocat](https://myoctocat.com/assets/images/base-octocat.svg)

The <img> element

Another way to display an image is to use the HTML <img> tag.

<img 
  src="https://myoctocat.com/assets/images/base-octocat.svg" 
  alt="Octocat"
/>

The <picture> element

To add a responsive image, like providing different images depending on whether the visitor is using light or dark mode, you can use the HTML <picture> tag.

<picture>
  <source 
      media="(prefers-color-scheme: dark)" 
      srcset="DARKMODE-IMG">
  <source 
      media="(prefers-color-scheme: light)" 
      srcset="LIGHTMODE-IMG">
  <img src="DEFAULT-IMG" alt="Octocat"/>
</picture>

Where you replace the capitalized titles in the srcset and src attributes, like DARKMODE-IMG or DEFAULT-IMG with the URL of the image you wish to use.

Creating code blocks

Fenced code blocks

You can create fenced code blocks in your writeup using three backticks ```.

```
  function test {
    console.log("Hello World!");
  }
```

Adding a line before and after your code block isn't required, but it makes the raw markdown file easier to read.

Syntax highlighting

You can enable syntax highlighting of your code block by including a language identifier in your code block.

```js
  function test {
    console.log("Hello World!");

Comments and horizontal rules

You can add comments to your writeup using the HTML comment syntax, like this:

<-- COMMENT -->

Replace the COMMENT with whatever note you wish to make to yourself, like adding a "to-do" item. The comment will be hidden in the output and only visible in your code.

You can also add horizontal rules to break up or divide your page content by typing three horizontal dashes.

---

This will generate a long horizontal line (a horizontal rule) on your page.

Remember that in HTML, the <hr> is a semantic element and represents a thematic break, or a transition between different topics or content. See the MDN article on the <hr> element to learn more.

Collapsing sections

If you want to keep your content visually clean while still including additional details or more information, you can use the <details> HTML tag to create collapsed sections.

<details>
  <summary>More information</summmary>  
  Here is some more info on the topic!
</details>
